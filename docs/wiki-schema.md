# join-quant 策略知识库 Schema

本文件定义 `wiki/` 策略知识库的结构与维护规则，灵感来自 Andrej Karpathy 的 LLM Wiki 模式：
**「源码是 raw 层，LLM 是维护者，wiki 是被维护的代码库」**。

人类负责：提供源码（由抓取流水线写入 `strategies/`）、提出问题、审阅概念页、裁决冲突。
LLM 负责：全部记账工作 —— 摘要、交叉链接、建索引、去重、标记矛盾、更新日志。

> 维护者绝不手写 wiki 正文；一律通过 `/ingest-strategy` 技能按本 Schema 生成与更新。

---

## 1. 目录结构

```
join-quant/
├── strategies/                 # raw 层：不可变源码（.py，带日期前缀与元数据头）
└── wiki/
    ├── index.md                # 目录：所有页面 + 一句话摘要 + 关键绩效
    ├── log.md                  # 追加式活动日志（ingest / query / lint）
    ├── strategies/             # 每个策略一页：<postId8>_<标题>.md
    ├── concepts/               # 跨策略的概念页：因子 / 择时 / 止损 / 仓位等
    └── authors/                # （可选）聚宽作者实体页
```

- raw 层 = `strategies/`，**永不修改**。wiki 引用它，不改写它。
- 所有 wiki 正文为**中文**，与源码及 `push-format.md` 保持一致。

## 2. 命名与标识

- **策略页文件名**：`<postId 前8位>_<安全标题>.md`，与源码 `.py` 的后缀 ID 对齐，便于互查。
- **概念页文件名**：规范概念名 + `.md`，使用下方受控词表，避免同义词分叉。
- **策略页内唯一标识**：以 `postId`（完整）为准；同一策略多个 `backtestId` 只建一页。
- **去重**：抓取阶段已按源码 SHA256 去重（见 `strategy-fetch.js`）。若 manifest 条目带 `duplicateOf`，**不建页**，仅在 `log.md` 记一行 `skip-dup`。

### 受控概念词表（canonical concepts）

`concepts` 与 `factors`（§2.1）是同一知识的两个层级：**factor 是具体信号（实例）；concept 是值得单独建页聚合的主题**。一个因子族复现到足够多策略时「升级」为 concept。

每个 concept 标注 `kind`：

- **因子族**：由某类选股因子升级而来（如 `小市值因子`←小市值信号）。
- **机制**：择时/风控/仓位类因子的聚合（如 `止损模块`、`仓位管理`）。
- **结构**：策略范式/标的类别，**不是**因子（如 `ETF轮动`、`网格交易`）。

新概念需先在此登记，再建页：

| 规范名 | kind | 同义/别名（归并到规范名） |
|--------|------|--------------------------|
| 小市值因子 | 因子族 | 微盘股、小盘、小市值轮动 |
| 动量与趋势 | 因子族 | 趋势跟踪、动量、量价 |
| 均值回归 | 因子族 | 回归、超跌反弹 |
| 择时-RSRS | 机制 | RSRS、阻力支撑相对强度 |
| 择时-均线 | 机制 | 均线择时、大盘择时、MA择时 |
| 止损模块 | 机制 | 均线死叉止损、追踪止损、阶梯止盈、风控止损 |
| 仓位管理 | 机制 | 等权配仓、仓位分配、资金管理 |
| ETF轮动 | 结构 | ETF动量轮动、基金轮动 |
| 行业轮动 | 结构 | 板块轮动、申万行业轮动 |
| 多因子模型 | 结构 | 多因子、截面因子、因子选股 |
| 打板与涨停 | 结构 | 打板、涨停、连板、龙头 |
| 龙虎榜 | 结构 | 龙虎榜选股、尾盘买入 |
| 可转债 | 结构 | 转债、双低 |
| 期货与套利 | 结构 | 股指期货、跨期套利、商品期货 |

### 因子 → concept 归并（rollup）

`因子族`/`机制` concept **由 factors 自动推导**，不必手填；只有 `结构` concept 需按策略范式手动标注。
ingest 时：先填 `factors`，再据下表补全对应 concept，最后手动加 `结构` concept。

| factor（角色·子类/信号） | → concept | kind |
|--------------------------|-----------|------|
| 选股·规模价值·小市值 | 小市值因子 | 因子族 |
| 选股·动量 | 动量与趋势 | 因子族 |
| 选股·（均值回归类信号） | 均值回归 | 因子族 |
| 择时·趋势均线 | 择时-均线 | 机制 |
| 择时·RSRS | 择时-RSRS | 机制 |
| 风控·任意非「无」 | 止损模块 | 机制 |
| 仓位·动态持仓数/流动性配权等 | 仓位管理 | 机制 |

> 并非所有 factor 都有 concept（如 `流动性`、`质量基本面` 暂无独立页）；也并非所有 concept 都是 factor（`结构` 类不是）。多个选股因子组合的策略可手动加 `多因子模型`（结构）。

### 2.1 因子分类（factors）

`concepts` 回答「属于哪些知识库主题」，`factors` 回答「用了哪些具体信号」，二者并存。
按**作用**分四类（与 `push-format.md` 的 选股池/仓位/止损 段对应）：

| 角色 | 作用 | 子类（受控） |
|------|------|--------------|
| **选股因子** | 选*哪些*标的 | 见下方选股子类受控表（必须归类） |
| **择时因子** | *何时*进出 / 调整暴露 | 趋势均线 / 位置估值 / 日历 / 防御信号 |
| **风控因子** | *何时*退出 / 保护 | 与 §3.1「止损类型枚举」对齐 |
| **仓位因子** | 配*多少* | 等权 / 动态持仓数 / 满仓单标的 / 流动性配权 / 预留资金 |

**选股子类受控表**（新增子类需先登记于此，类比概念词表）：

| 子类 | 含义 | 示例信号 |
|------|------|----------|
| 规模价值 | 市值 / 估值 / 折溢价 | 小市值、PE/PB 过滤、ETF 折价率 |
| 质量基本面 | 财务质量过滤 | 国九条（净利>0/营收>1亿）、三正（pb>0/inc_return>0/ocf>0） |
| 动量 | 价格趋势强度 | 对数回归动量、涨速 |
| 技术量价 | 技术指标 / 量价形态 | 涨停基因、底部放量、均线乖离、RSI/MACD/KDJ/布林、低开剥头皮 |
| 流动性 | 成交活跃度过滤 | 买一档金额、成交额门槛 |
| 情绪事件 | 事件 / 资金 / 情绪 | 龙虎榜、连板、资金流入 |

**风控因子枚举**：`无` / 止损线 / 止盈倍数 / 大盘趋势 / 盈利保护 / 组合回撤 / 排名止损 / 过热切债 / 破开盘价 / 最大持仓天数。

frontmatter 记录格式（选股按子类嵌套，其余角色为列表）：

```yaml
factors:
  选股:
    规模价值: [小市值]
    质量基本面: [国九条]
    技术量价: [涨停基因, 底部放量]
  择时: [深证综指MA10偏离, 1/4月日历空仓]
  风控: [止损线-9%, 大盘趋势, 止盈×2]
  仓位: [等权, 动态持仓数(MA偏离3~6)]
```

未用到的角色省略该键；具体信号名为描述性文字（子类标签才受控）。

### 3.1 策略页 `wiki/strategies/<postId8>_<标题>.md`

```markdown

backtestId: <backtestId>

sourceFile: strategies/<带日期前缀的文件名>.py
joinquantPost: <聚宽原帖链接，若源码头中有>
author: <作者，若有>
concepts: [小市值因子, 择时-均线]      # 因子族/机制 concept 由 factors 自动归并；仅 结构 concept 需手填
factors:                               # 本策略用到的具体信号，按 §2.1 四角色分类（concept 的实例来源）
  选股:
    规模价值: [小市值]
    质量基本面: [国九条]
  择时: [深证综指MA10偏离, 1/4月日历空仓]

  仓位: [等权, 动态持仓数(MA偏离3~6)]

stats: { annualReturn: <小数>, sharpe: <数>, maxDrawdown: <小数>, periodLabel: <YYYY-MM> }

**一句话**：<用一句话概括该策略做什么>

## 忠实翻译

**选股池**：…
**仓位分配**：…
**止损**：…（仅当代码中有此模块；被注释掉的不写）
**风控**：…（含滑点、手续费、回测区间、代码行数）

## 绩效
📊 年化XX.X% | 夏普X.XX | 最大回撤XX.X% | 回测区间 YYYY-MM 起

## 涉及概念
```

### 3.2 概念页 `wiki/concepts/<规范名>.md`

概念页是知识库的**核心价值**：横向聚合所有相关策略，提炼共性与差异。

```markdown
---
concept: <规范名>
kind: <因子族 | 机制 | 结构>        # 见 §2 受控概念词表
rollup: <因子族/机制时填对应 factor，如 选股·规模价值·小市值；结构留空>
aliases: [<别名…>]
strategyCount: <N>
updatedAt: <YYYY-MM-DD>
---

# <规范名>

**定义**：<一段话解释该因子/技术在量化中的含义>

## 共性做法
<这些策略在该概念上的通用实现：股票池、参数区间、调仓节奏等>

## 变体与差异
「因子构成」列为各策略 `factors` 的角色化压缩签名（`选:… ｜ 择:… ｜ 控:… ｜ 仓:…`，无该角色则省略），便于横向对齐。
**该列由 `node utils/wiki-factor-signature.js <策略页>` 从 `factors` frontmatter 生成，请勿手写**（lint 会校验漂移）。

| 策略 | 因子构成（选/择/控/仓） | 选股池/数量 | 与众不同之处 | 年化/回撤 |
|------|------------------------|------------|--------------|-----------|
| [[<postId8>_<标题>]] | 选:小市值·国九条质量 ｜ 控:止损线-9%+大盘趋势 ｜ 仓:等权·动态3~6 | 深证综指 / 3~6 | … | XX% / XX% |

## 绩效横评
<按年化/夏普/回撤对比，标注表现最好的版本及其回测区间；声明为「截至 <date> 的快照」>

## 关联概念
- [[择时-均线]]、[[仓位管理]] …

## 待研究 / 空白
<该概念下尚缺的实验、数据缺口>
```

## 5. log.md 格式

追加式，便于 `grep`。每行一条：

```
## [YYYY-MM-DD] <op> | <说明>
```

`op` 取值：`ingest`（建/更新策略页）、`skip-dup`（命中去重）、`concept`（新建概念页）、`query`（问答并回填）、`lint`（健康检查）、`merge`（概念归并）。

示例：
```
## [2026-06-27] ingest | 60倍七星高照 (965618d9) → concepts: ETF轮动
## [2026-06-27] skip-dup | 低开小市值副本 (1b5ae4fe) dup-of 28d7e5c3
## [2026-06-27] concept | 新建 小市值因子（6 篇）
```

## 6. Ingest 工作流（每篇策略）

1. 读取源码 `strategies/<file>.py` 与 manifest 中对应 `stats`。
2. 若 manifest 条目带 `duplicateOf` → 记 `skip-dup`，不建页，结束。
3. 按 §3.1 写策略页；忠实翻译段严格套用 `push-format.md`。
4. 判定其涉及的**规范概念**（§2 词表）；新概念需先登记词表再建概念页。
   - 若该策略的翻译已产出「概要要素行」（见 `push-format.md` §2，含 基准/持股/调仓/止损/因子），**优先直接复用**其字段填充 frontmatter，避免从源码二次推断；`因子[...]` 即 frontmatter 的 `concepts`。
5. 对每个涉及概念：**追加**该策略到概念页的「变体与差异」「绩效横评」，**不覆盖**已有内容；刷新 `strategyCount` 与 `updatedAt`。
6. 更新 `index.md`（策略节 + 受影响概念节）。
7. 追加 `log.md`。
8. 发现与概念页或既有策略的矛盾 → 写入该策略页「矛盾」小节并在 `log.md` 标注，留待人工裁决，**不自行删改既有结论**。

## 7. Query 工作流

- 经 `index.md` 定位相关页 → 阅读 → 带引用作答（引用 = `[[页名]]` 或 `strategies/<file>.py`）。
- 有价值的结论回填为概念页的小节或新页，并记 `log.md`。

## 8. Lint 工作流（定期）

扫描并在 `log.md` 报告（不自动改动结论）：
- 孤儿策略页（无任何 `[[概念]]` 链接）
- 词表中已出现但尚无概念页的概念
- 概念页「绩效横评」中过期的「最佳版本」声明
- 双向链接缺失（策略页引用了概念，但概念页未收录该策略）
- **因子构成漂移**：概念页表中的「因子构成」单元格与 `node utils/wiki-factor-signature.js <策略页>` 的输出不一致（应始终由该脚本从 `factors` frontmatter 生成）

---

## 9. 不可违反的原则

- **raw 不可变**：绝不修改 `strategies/` 下的任何文件。
- **追加优先**：更新概念页时追加新信息，保留历史；版本演进交给 git。
- **可溯源**：每条结论可回溯到具体策略页或源码文件。
- **人裁冲突**：矛盾只标记、不自行裁决。
- **受控命名**：概念一律走 §2 词表，先登记后建页，杜绝同义分叉。
