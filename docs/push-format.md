# JoinQuant 策略推送格式规范

本文档定义每日抓取的量化策略推送到微信的格式标准。

---

## 推送结构

每条策略推送由以下部分组成，按顺序排列：

### 1. 标题行
```
📌 *策略标题*
```

### 2. 概要要素行（结构化，必填）

一行用「字段[值]」表达策略的可聚合要素。它**同时**用于微信推送的速览，以及 `wiki/` 知识库 ingest 时直接填充策略页 frontmatter（`concepts` + `factors` + 基准/持股/调仓），无需再从源码二次推断。

按**因子作用**拆分 选股/择时/风控 三类信号（仓位由 持股/调仓 隐含），末尾附受控概念标签：

```
概要：基准[XXX] | 持股N | 调仓[每日/每周W/每月D HH:MM] | 选股[信号…] | 择时[信号…] | 风控[类型…] | 概念[概念1,概念2]
```

字段说明：

| 字段 | 取值 | 说明 |
|------|------|------|
| 基准 | 指数/ETF 名或代码 | `set_benchmark` 的标的（如 深证综指/国证2000/沪深300） |
| 持股 | 整数或区间 | 目标持仓数；动态调仓写区间（如 `3~6`） |
| 调仓 | 频率 + 时间 | 如 `每日 9:30`、`每周三 9:31`、`每月首日` |
| 选股 | 选股因子，逗号分隔 | 具体信号；归入 `wiki-schema.md` §2.1 选股子类（规模价值/质量基本面/动量/技术量价/流动性/情绪事件） |
| 择时 | 择时因子 | 趋势均线 / 位置估值 / 日历 / 防御信号；无则省略 |
| 风控 | 风控因子枚举 | 见下；多个用 `+`，无则 `无` |
| 概念 | **受控概念**，逗号分隔 | **必须**取自 `wiki-schema.md` §2 受控概念词表（不得自造同义词：微盘股→`小市值因子`） |

**风控因子枚举**（与 [[止损模块]] 概念页对齐）：`无` / `止损线` / `止盈倍数` / `大盘趋势` / `盈利保护` / `组合回撤` / `排名止损` / `过热切债` / `破开盘价` / `最大持仓天数`。

> 完整的四角色 `factors` 结构（含选股子类嵌套与 仓位 角色）以 `wiki-schema.md` §2.1 为准；本行是其面向人读的扁平化速览。

格式示例：
```
概要：基准[深证综指] | 持股3~6 | 调仓[每周二 10:00] | 选股[小市值,国九条质量] | 择时[MA10偏离,1/4月空仓] | 风控[止损线-9%+大盘趋势+止盈×2] | 概念[小市值因子,择时-均线,止损模块]
```

### 3. 忠实翻译（核心内容）

忠实翻译是根据策略 Python 源码逐段解析后生成的自然语言描述，要求**如实反映代码逻辑**，不添加实现细节、注释或个人解读。

每段以 `选股池：`、`仓位分配：`、`止损：`、`风控：` 开头，段间空一行。

#### 选股池
- 描述选股时间、股票池范围、剔除条件、市值排序规则
- 格式示例：每天 09:00，在「上证综指 + 深证综指」（约4000只）内进行筛选。先剔除近63个交易日内有停牌的股票，再从中选取总市值最小的20只股票，构建次日买入候选列表。

#### 仓位分配
- 描述等权配仓逻辑、调仓时间、买卖规则
- 只描述实际执行的逻辑，不描述实现细节（如某个循环写了两遍这类内容不写）

#### 止损（仅当代码中有此模块时）
- 描述触发条件（均线死叉等）、执行动作、清仓条件
- **必须使用代码中实际的均线周期**（如 MA10、MA50），不能硬编码默认
- 未启用、被注释掉的止损模块**不写入描述**

#### 风控
包含以下内容（按实际代码情况）：
- 停牌过滤规则（如有）
- 滑点设置
- 手续费（含印花税细节，如「买入万3，卖出千1.3，含印花税千分之一」）
- 回测区间（**必须**：来自 stats API 的 `period_label`，格式「YYYY-MM」）
- 代码行数（**必须**）

#### 以下内容**不应出现**：
- 未启用的函数（如被注释掉的 jieti 阶梯止盈）
- 冗余代码提示（如「循环写了两遍」）
- 实现细节、个人解读、修改建议

### 4. 绩效数据
```
📊 年化XX.X% | 夏普X.XX | 最大回撤XX.X%
```
若绩效未公开，显示「📊 绩效未公开」。

## 实现约束

### 均线周期默认值
若代码中 `g.short_d` / `g.long_d` 参数无法解析，使用以下默认值：
- `short_d` → 默认 10（MA10）
- `long_d`  → 默认 50（MA50）

**禁止**使用 `|| 2` / `|| 20` 等不反映实际代码的默认值。

### 回测区间数据来源
优先使用 `/algorithm/backtest/stats` API 返回的 `period_label` 字段（格式：`YYYY-MM`），表示回测开始月份。

### 推送目标
每条推送针对**一个 postId**（一个策略帖子），即使该帖子有多个 backtestId 也只推送一次（取 score 最高的版本）。
