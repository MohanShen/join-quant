# 解剖报告：052da9b4_市值选股+阶梯调仓（DQ — 为什么不行）

## 一句话结论
一个**每日全额调仓 smallest-20 市值股**的策略：每日重排（上证综指+深证综指）最小市值前 20 只、等额重配整仓，叠加一个有 bug 的「阶梯调仓」止盈/加仓层。在冻结成本 harness 下 **DQ 灾难性亏损**——train 2022-2023 收益 **−81.62%**、年化 −57.18%、Sharpe −1.84、**maxDD 82.9%**。头条「年化42%」是短窗/忽略换手成本的幻觉。

## 为什么不过门槛（DQ 归因）
1. **每日全额 churn = 极端换手成本出血**：`rebalance()` 每个交易日把整仓卖到新的 smallest-20 并等额重买。smallest-cap 上每日全额换手，在冻结佣金（买万3 / 卖万3+印花税千1）下每次往返 ~0.16% 成本，日复一日复利吞噬本金——这是 DQ 的**主因**。
2. **无风控 / 无择时**：满仓 smallest-20，无止损、无空仓、无防御。2022 熊市 + 微盘踩踏直接击穿 → 82.9% maxDD（近乎腰斩再腰斩）。
3. **阶梯调仓逻辑有 bug**：止盈 `if ret>0.05: 卖半 elif ret>0.1 ... elif ret>0.15 ...`——后两个 elif **永不可达**（ret>0.05 已先命中）；加仓 `if MA10>price: 加20% elif MA15>price ...` 在价格跌破均线时**不断加仓下跌标的**（越跌越买），放大亏损。逻辑本身也在助推灾难。

## 机理与真实性
- ⚠ 本策略是**真·负 EV**（不是零滑点高估的假象）：即便零滑点，光是佣金+印花税的每日全额换手就足以在 smallest-cap 上造成灾难性出血；−81.62% / 82.9% maxDD 在任何合理成本下都不可能是可部署策略。方向确定（DQ 真实），confidence=high。

## 待研究 / 反哺 auto-research
- **反哺**：`smallest-cap × 每日全额调仓` 是**成本自杀**——微盘的 edge 必须配**低换手**（周/月频）才可能兑现；日频全仓 churn 把 size 因子的 alpha 全部（且更多）交给交易成本。ideator 生成微盘点子时应把换手频率作一等约束（cf. [[小市值因子]] 家族的周频调仓 + 空仓期是可行版本）。
- 「越跌越加仓 + 无止损」是回撤放大器；DQ 策略的典型死法。
