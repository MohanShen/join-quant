# 解剖报告：bf30eb36_三马105五福35修复

## 一句话结论
一个 191KB / 4460 行的多策略 MERGE（三马 v10.5 + 五福 v3.5 + 更多），跑 isolated 虚拟子账户上的 **50% 小市值 + 50% 五福 ETF-momentum 轮动** blend（`g.portfolio_value_proportion = [0.5, 0, 0.5, 0]`）。基线（train 2022-2023，冻结零滑点）total 145% / 年化 56.62% / **Sharpe 2.57** / maxDD 9.92% / **obj 0.467 = MARGINAL gate-pass**（Sharpe 刚过 2.5）；其低 DD 迁移自小市值家族的 blackout 驱动低回撤。**注：该策略回测过慢无法在 harness 下消融**（sleeve-attribution 变体两次 >30min slow-skip），故 sleeve 拆分为**推断而非实测**（LIKELY 是五福 momentum sleeve 稀释强小市值 sleeve，平行 [[9f6f8188_大小盘多策略]] / [[4fa17009_五福v5差异化滤波]]）。⚠ 零滑点高估 + 边际 Sharpe → live 很可能脆弱。

## 机理与真实性
- **边际 gate-pass**：Sharpe 2.57 刚过 2.5 门槛、obj 0.467，远低于纯小市值家族研究的 ~4-6 Sharpe；blend 把强小市值 sleeve 拉到边际，与「五福 momentum sleeve 稀释」的叙事一致（但未实测）。
- **⚠ 零滑点高估**：小市值 + ETF momentum 均高换手，冻结零滑点回测显著高估可实现收益；叠加边际 Sharpe，live 很可能脆弱。
- **maintainability / realism 红旗**：191KB / 4460 行、每根 bar 重跑 250 日 ETF 预载 + 全池 RSRS（0 权重 sleeve 也照跑 run_daily）→ 验证成本极高、迭代不现实。这类重量级 merge **难以 backtest / ablate / validate**，且易维护出错（cf 姊妹 [[2a4882de_动态仓位小市值五年20倍]]：头条特性静默损坏、打开即 crash）。

## 待研究 / 反哺 auto-research

**反哺 auto-research**：
1. **重量级多策略 merge 可能不可实用地 backtest / ablate / validate** —— 每根 bar 的重预载使其在 harness 下超时，是一个 **maintainability 红旗**；ideator/critic 应对 >100KB 多 sleeve merge 打折（验证成本 + 维护出错风险，cf 2a4882de 头条特性静默损坏）。
2. **强小市值 + 脆弱 momentum sleeve 的边际-Sharpe blend 很可能稀释**（平行 [[9f6f8188_大小盘多策略]] 的实测：多样化 sleeve 是纯净拖累）——优先**纯强 sleeve** 而非 naive 拼装；边际头条 obj 常是 blend-weight 产物而非组件本身。
